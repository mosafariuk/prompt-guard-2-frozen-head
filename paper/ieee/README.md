# Building the IEEEtran PDF

The single-file source of truth is `../manuscript.md`. This directory holds the
IEEEtran wrapper (`main.tex`). The body is produced **mechanically** from the
markdown so math, tables, and proofs convert faithfully — do not hand-edit a
duplicated body.

## Prerequisites (not present in the authoring environment)

- `pandoc` (markdown → LaTeX)
- A LaTeX distribution with `IEEEtran.cls` (TeX Live: `texlive-publishers`)

## Build

```bash
cd paper/ieee

# 1. Convert the body (everything after the abstract). Strip the front-matter and
#    the per-section provenance blocks for camera-ready if desired.
pandoc ../manuscript.md \
  --from gfm+tex_math_dollars \
  --to latex \
  --top-level-division=section \
  -o body.tex

# 2. (Optional) convert references. For a quick pass, keep \input; for a proper
#    build, hand-author refs.bib from ../10_references.md and enable \bibliography
#    in main.tex.

# 3. Compile.
pdflatex main.tex && pdflatex main.tex
```

## Notes / expected manual fixes

`pandoc` gets ~95% of the way; a two-column IEEEtran build typically needs a few
manual touches, none structural:

- **Wide tables** (STRIDE Table A.1, resource-envelope Table II) may overflow a
  column — wrap in `table*` (full-width) or shrink with `\footnotesize` / `\resizebox`.
- **Fenced code blocks** (Fig. 1 dataflow, SQL) become `verbatim`/`lstlisting`; check
  they fit the column width.
- **Section numbering**: the markdown uses explicit `# I.`, `# II.` … headings.
  Either keep them literal or let `\section` auto-number (delete the roman numerals
  from the headings before conversion).
- **Provenance blocks** (`> Evidence status: …`) become block quotes; delete for
  camera-ready.

The abstract is already inline in `main.tex` (kept authoritative there); pandoc's
converted abstract from the markdown front-matter should be dropped.

## Why this path

Neither `pandoc` nor `pdflatex` was available where this repo was authored, so a
hand-written 1600-line `.tex` could not be compiled or verified. Rather than ship an
unverifiable LaTeX conversion, the wrapper here is small and correct, and the body is
generated by a deterministic tool you can run and inspect. This keeps the same
"nothing asserted that wasn't checked" bar as the rest of the project.
